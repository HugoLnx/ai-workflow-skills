# cross-skills

A meta-skill for generating, translating, validating, and maintaining AI coding
assistant configuration files across multiple agent ecosystems.

## Project Overview

`cross-skills` is a Claude Code skill that acts as a universal configuration manager
for AI coding assistants. It uses a **Universal Intermediate Representation (UIR)**
as an ecosystem-agnostic source of truth, then generates or translates configurations
for Claude Code, Cursor, OpenAI Codex CLI, and Aider.

**Core concept**: Instead of maintaining separate CLAUDE.md, .cursor/rules/*.mdc,
AGENTS.md, and CONVENTIONS.md files independently, you maintain one `uir.json` and
let cross-skills generate all the ecosystem-specific files from it.

## Directory Layout

| Path | Purpose |
|---|---|
| `skill.md` | Claude Code skill entry point — mode dispatch + project references |
| `modules/` | Composable instruction modules (one per operating mode) |
| `schema/` | UIR JSON schema, capability matrix, and an annotated example UIR |
| `templates/` | Per-ecosystem config file templates with `{{PLACEHOLDER}}` variables |
| `validation/` | Validation rules (machine-readable JSON) and gap analysis reference |
| `examples/` | Worked end-to-end examples showing UIR → generated configs |

## Operating Modes

Invoke with the Skill tool as `cross-skills`. Five modes:

| Mode | Trigger | What it does |
|---|---|---|
| `analyze` | Point at a repo | Scans repo, produces `uir.json` |
| `generate` | Have a UIR | Produces all ecosystem config files |
| `translate` | Have existing configs | Converts ecosystem A → UIR → ecosystem B |
| `validate` | Have configs or UIR | Checks for errors, conflicts, gaps |
| `build-skill` | Need a new skill | Generates a skill file for one or more ecosystems |

## Key Files

- `schema/uir.schema.json` — The UIR JSON Schema (draft-07). All fields documented with descriptions.
- `schema/capability-matrix.json` — Machine-readable ecosystem capability matrix. Used by the validator.
- `schema/uir.example.json` — Annotated UIR for a React/TS + Node.js project. Use as a reference.
- `modules/analyzer.md` — Instructions for `analyze` mode
- `modules/generator.md` — Instructions for `generate` mode
- `modules/translator.md` — Instructions for `translate` mode
- `modules/validator.md` — Instructions for `validate` mode
- `modules/builder.md` — Instructions for `build-skill` mode
- `validation/gap-analysis.md` — Human-readable translation degradation reference

## Supported Ecosystems

| ID | Tool | Config files | UIR fidelity |
|---|---|---|---|
| `claude-code` | Claude Code | CLAUDE.md, AGENTS.md, settings.json, `.claude/skills/` | Full |
| `cursor` | Cursor | .cursor/rules/*.mdc | High (permissions/hooks degrade) |
| `codex` | OpenAI Codex CLI | AGENTS.md, system-prompt.txt | Medium (scoping lost, hooks dropped) |
| `aider` | Aider | .aider.conf.yml, CONVENTIONS.md, .aiderignore | Medium (scoping/hooks lost) |

**Extensible to**: windsurf, roo-code, continue, cline, openhand, gemini-cli.
See `schema/capability-matrix.json` for how to add a new ecosystem.

## AI Behavior Rules

When working in this project:
- The templates in `templates/` use `{{PLACEHOLDER}}` syntax — do not confuse with real content
- All generated files must include the `# Generated by cross-skills` header comment
- Never modify `schema/uir.schema.json` to work around a constraint — update the schema with proper versioning
- The `validation/rules.json` is the authoritative source for validation rule IDs and severities
- When adding a new ecosystem: follow the checklist in `schema/capability-matrix.json extensibility_notes`
