# Rules Frontmatter Expert

Supported fields per harness for rules. Unlike skills, rules use a sidecar `.yml` file rather than per-harness frontmatter files inside the source.

---

## Sidecar Format: `.ai/rules/<name>.yml`

Controls how each harness consumes the rule. Optional — a rule with no `.yml` sidecar is always-on for all harnesses.

```yaml
cursor:
  description: "Short description shown in Cursor's rule picker UI"
  globs: "src/api/**/*.ts,app/api/**/*.ts"
  alwaysApply: false
copilot:
  applyTo: "src/api/**,app/api/**"
```

### Cursor sidecar fields

| Field | Required | Notes |
|---|---|---|
| `cursor.description` | yes (if cursor section present) | Shown in Cursor's rule picker UI |
| `cursor.globs` | no | Comma-separated glob patterns; omit for always-on |
| `cursor.alwaysApply` | no | Default `false`; set `true` for project-wide always-on rules |

### Copilot sidecar fields

| Field | Required | Notes |
|---|---|---|
| `copilot.applyTo` | no | Comma-separated globs for Copilot's `applyTo` field; omit for repo-wide rules |

## Harness Output Formats

### Cursor: `.cursor/rules/<name>.mdc`

```markdown
---
description: API layer conventions for route handlers
globs: src/api/**/*.ts,app/api/**/*.ts
alwaysApply: false
---

<!-- generated by build-rules script — edit .ai/rules/<name>.md to update -->

<verbatim content of .ai/rules/<name>.md>
```

If no sidecar or `globs` is blank and `alwaysApply` is not set: emit `alwaysApply: true` and omit `globs`.

### GitHub Copilot: `.github/instructions/<name>.md`

```markdown
---
applyTo: "src/api/**,app/api/**"
---

<!-- generated by build-rules script — edit .ai/rules/<name>.md to update -->

<verbatim content of .ai/rules/<name>.md>
```

If no `copilot.applyTo` in sidecar: omit the frontmatter block entirely.

### Claude Code / Codex: Managed Imports Block

Rules are imported via a managed block inserted into `CLAUDE.md` and `AGENTS.md`:

```
<!-- cross-rules:imports:start -->
@file .ai/rules/rule-name.md
<!-- cross-rules:imports:end -->
```

This block is always updated as a unit; content outside the delimiters is never touched. Claude Code and Codex have no per-rule frontmatter — the import mechanism handles routing.

---

## No-Sidecar Default Behavior

| Harness | No sidecar behavior |
|---|---|
| Claude Code | Imported in managed block — always applied |
| Codex CLI | Imported in managed block — always applied |
| Cursor | `alwaysApply: true`, no `globs` |
| GitHub Copilot | No frontmatter block — applies to all files |
