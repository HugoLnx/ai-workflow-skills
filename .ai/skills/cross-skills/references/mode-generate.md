# Mode: generate

**Triggers**: "generate a skill", "create a skill for X", "new skill", "add a skill"

**Goal**: create a complete new skill under `.ai/skills/<name>/`.

---

## Quality Standard

Every skill generated by this mode must pass the 10-axis checklist in `references/skill-quality-checklist.md` before output is finalized. Read that file now if you have not already.

---

## Steps

1. **Gather information** — ask the user for (if not already provided):
   - Skill name (kebab-case slug, e.g. `api-conventions`)
   - What the skill does and when it should trigger (include examples of triggering phrases)
   - Which file globs it applies to, or "all files"
   - Which tools the skill needs (`Read`, `Write`, `Bash`, `WebSearch`, etc.)

2. **Create `content.md`** at `.ai/skills/<name>/content.md` — pure markdown, no frontmatter. Include:
   - One-sentence purpose statement with an `as of [YYYY-MM-DD]` marker on the same line
   - **When to use** (✅) and **when NOT to use** (❌) sections with specific trigger keywords and explicit exclusions — both sections are required
   - Step-by-step process or decision tree; use a Mermaid `flowchart TD` for every decision flow (required for any branching logic — not optional)
   - **At least 3 anti-patterns** using the full 5-field template from `references/skill-quality-checklist.md` (Novice / Expert / Timeline / LLM-mistake / Detection)
   - A References table if the skill has sub-files — every row must have a one-line "Consult When" description
   - Keep `content.md` under 500 lines — move deep reference material into `references/` files

3. **Write the description** for each `.yaml` file using this formula:

   ```
   [What it does] [When to use it] [Trigger keywords]. NOT for [Exclusions].
   ```

   Constraints:
   - 25–50 words ideal, 100 words maximum
   - Must include at least one concrete NOT clause — "NOT for [vague topic]" does not count
   - The description must appear in all four `.yaml` files (adapted per harness, semantically identical)

   Example:
   > Generate, translate, validate, and maintain AI coding-assistant skills across Claude Code, Cursor, Codex, and Copilot from a single source of truth under .ai/skills/. Use when generating configs, translating between harnesses, validating skill folders, or analyzing a codebase. NOT for CLAUDE.md, AGENTS.md, or any project-level harness config.

4. **Create all four `.yaml` files** inside `frontmatter/` at `.ai/skills/<name>/frontmatter/` — consult `references/harness-frontmatter.md` for the exact fields and format per harness.

5. **Create `CHANGELOG.md`** at `.ai/skills/<name>/CHANGELOG.md`. Minimum content:

   ```markdown
   # Changelog

   ## [0.1.0] — YYYY-MM-DD
   ### Added
   - Initial skill: [one-line description of what it does]
   ```

   Replace `YYYY-MM-DD` with today's date. The file is symlinked into harness folders like any other non-yaml file.

6. **Define output contracts** for each mode or major action the skill performs. For each distinct output, include in `content.md` a markdown template with `<placeholder>` slots:

   ```markdown
   ## Output: [Mode or Action Name]

   **Result**: <summary of what was done>
   **Files created**:
   - `<path>`: <purpose>
   **Next step**: <what the user should do next>
   ```

   Also document edge cases: what happens when input is missing, a step fails, or the output already exists.

7. **Describe the build step** — explain how to run `.ai/build-skills.ps1` (Windows) or `.ai/build-skills.sh` (macOS/Linux) to propagate the new skill into all four harness folders. Offer to produce a build script if one does not exist yet (invoke `build-skill` mode).

8. **Run the pre-publish gate** from `references/skill-quality-checklist.md` before finalizing:
   1. Word-count the description — trim if >100 words
   2. Count anti-patterns — must be ≥3 with all 5 fields
   3. Confirm every decision flow has a Mermaid diagram
   4. Confirm `CHANGELOG.md` was created with a dated entry
   5. Confirm `as of [date]` marker is in `content.md`

   Then **print the creation summary**:
   ```
   Created .ai/skills/<name>/content.md                    (as of YYYY-MM-DD)
   Created .ai/skills/<name>/CHANGELOG.md
   Created .ai/skills/<name>/frontmatter/claude.yaml
   Created .ai/skills/<name>/frontmatter/codex.yaml
   Created .ai/skills/<name>/frontmatter/cursor.yaml
   Created .ai/skills/<name>/frontmatter/copilot.yaml

   Quality gate: [✓ passed / ✗ N issues — fix before publishing]

   Run .ai/build-skills.ps1 (Windows) or .ai/build-skills.sh (macOS/Linux)
   to propagate to harness folders.
   ```
